# GM 裁定クイックリファレンス v1.0

**用途**: M0 セッション中、GM(ファシリテータ)が手元で参照する1枚もの。[m0_protocol_v1.md §2.4](./m0_protocol_v1.md) が指示する「A4両面印刷用」の現物。

**印刷想定**: A4 × 1枚(両面推奨)。GMは全6セッションで同一人物 → この1枚で裁定のブレをゼロにする。

---

## A. 1ターンの進行(約2–3秒圧縮)

1. **被験者**: 最大2アクションを宣言(例: 「左手で右袖を狙う、強度2」「腰を右に切る +1」)
2. **GM**: §B でグリップ試行を裁定、その結果をボードに反映
3. **GM**: §C でAI反応を1つだけ選んで宣言
4. **GM**: §E で状態自然変化を適用
5. **GM**: §D で判断窓発火チェック
6. 判断窓発火 → 被験者に「選ぶ / 見送る」を問う

---

## B. グリップ試行の裁定([m0_protocol_v1.md §4.1](./m0_protocol_v1.md))

被験者が「(手)で(部位)を狙う、強度N」と宣言したら:

| 状況 | 宣言 | ボード操作 |
|---|---|---|
| ターゲット空き + 相手同所防御なし | **「掴めた」** | 赤丸をゾーン、強度マーカーN |
| 相手が同ゾーンを防御中 | **「弾かれた、手が戻る」** | 赤丸ニュートラル / §C.2 記憶 |
| 直近1ターン以内に同手・同ゾーンで弾かれ | **「もう一度弾かれる」** | 同上 |

### B.1 短期記憶(§4.2)

弾かれた直後の1ターン、**同じ手 × 同じゾーン**の再試行は無効。チェックマーク or サイコロで記録。逆の手で同じゾーンや、同じ手で別ゾーンは即座にOK。

---

## C. AI反応の選択([m0_protocol_v1.md §4.3](./m0_protocol_v1.md))

GMは毎ターン1つだけ宣言。優先度は上から:

1. **判断窓条件を1つ崩しに行く** — 被験者が両袖握り → 1本を切りに行く
2. **自分の崩れを戻す** — 姿勢崩し値 ≥ 3 → -1、被験者の腰角度を 0 に押し戻す
3. **パスを試みる** — 5ターン経過 or 姿勢崩し値 0 が 3ターン継続 → パス宣言 = セッション終了

---

## D. 判断窓の判定([m0_protocol_v1.md §5](./m0_protocol_v1.md))

### D.1 発火チェック

毎ターン終了時、**6技カード全部**を上から照合。1つでも全条件成立 → **「判断窓が開きました」** と宣言し、成立した技カードを被験者前に並べる。

**複数技同時成立**: すべて並べて選ばせる。これ自体が観察ポイント。

### D.2 応答の扱い

| 応答 | GM処理 |
|---|---|
| 技を選ぶ | 「技が極まりました」→ 中断 → 即時Q([interview_sheet](./interview_sheet_v1.md) §7.1) |
| 見送る | 窓を閉じる、**次ターン1回は発火しない**(クールダウン) |
| 5秒無回答 | 見送り扱い |

### D.3 連発抑制

見送りの次ターンは技カード確認をスキップ。入力系v1.0 §D.3 の 400ms クールダウンに相当。

---

## E. 状態の自然変化(ターン終了時、§4.4)

- 被験者が腰を同方向に `±1` 以上で **3ターン連続** 切り続けた → 姿勢崩し値 `+1`
- 被験者が **グリップなし**が 2ターン続いた → 姿勢崩し値 `-1`
- 被験者が「**呼吸を整える**」と宣言 → 次ターンの行動権 -1(スタミナ系は演出のみ)
- **腕が引き出された**(三角絞めの前提、技カード③参照):
  - 袖 or 手首を強度 ≥ 2 で掴んだまま **3ターン連続で引き続けた** → 「引き出された」状態 ON
  - その後グリップが切れる or GMが「相手がベースを取り戻した」宣言 → OFF

---

## F. セッション終了条件

| 条件 | 終了理由 |
|---|---|
| 被験者が技を選び成立 | **技極まる** |
| GMがパス宣言(§C.3) | **パスされる** |
| 判断窓発火が3回起きた | **時間切れ(達成)** |
| 7分経過 | **時間切れ(未達成)** |

終了したら [session_log_template_v1.md](./session_log_template_v1.md) の終了理由欄に記入し、即時Qインタビュー([interview_sheet_v1.md](./interview_sheet_v1.md))に移る。

---

## G. よくある境界ケース

| ケース | 処理 |
|---|---|
| 被験者が「強度1.5」など中間値を宣言 | 「0 / 1 / 2」の3段階から最も近い側へ丸める、迷ったら低い側 |
| 判断窓発火中に条件が崩れた | **即時に窓を閉じる**(入力系v1.0 §D.2 "条件崩れで強制 CLOSING")。「条件が崩れたので窓が閉じました」と宣言 |
| 被験者が「両手同時に違うゾーンを狙う」 | OK。各手を独立に§B.1で裁定。ただし宣言アクション数は2まで |
| 被験者が技の条件を自分で読み上げて確認したい | OK。カードは伏せないし参照自由 |
| GMが裁定に迷った | その場で **「保留」と宣言 → ターン終了後に §E を適用してから再判定**。裁定に明らかに誤りがあったら「訂正」と宣言して戻す。**ただしこのことは後で記録する** |
