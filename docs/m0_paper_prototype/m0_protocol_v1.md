# M0 ペーパープロトタイプ プロトコル v1.0

**プロジェクト**: 柔術シミュレータ
**目的**: 入力系v1.0で定義した「判断空間」が現役柔術家にとって本物と感じられるかを、コード実装前に検証する
**前提文書**: [入力系詳細設計書 v1.0](../design/input_system_v1.md), [Visual Pillar Document v1.0](../visual/Visual_Pillar_Document_v1.docx)
**版**: 2026-04-18

---

## 0. M0ゲート（本文書で定義）

READMEの「5人中4人」基準を本被験者構成（6名）に合わせて再定義する:

> **青帯3/3 かつ 白帯2/3 が「判断空間は本物」と認めること。**

**根拠**:
- 青帯は判断空間に対する解像度が高い。ここを満点(3/3)で取れない設計は M1 以降の信頼を失う
- 白帯は「学習者として違和感がないか」のサニティチェック。2/3で十分
- 単純な総数ベース(例: 5/6)では青帯不合格 + 白帯満点でも通ってしまうため、帯別ゲートを採用

**「判断空間が本物」の操作的定義**: §6.2の判定基準を満たすこと。

## 1. プロトタイプ形式

### 1.1 ハイブリッド構成

| 要素 | 担当 | 媒体 |
|---|---|---|
| 状態の可視化（フック、グリップ、姿勢崩し値） | 物理ボード上のトークン・カード | 紙 |
| 入力の宣言（被験者の意図） | 被験者が口頭で宣言 | — |
| 結果の裁定（弾き判定、判断窓発火） | GM（ファシリテータ）が判定表に従い裁定 | 進行表 |
| 相手プレイヤー（AI役） | GMが演じる | — |

### 1.2 「探り合い」の紙上での扱い

入力系v1.0のグリップファイトは1秒未満のリアルタイム攻防だが、紙では時間圧縮できないため次のように抽象化する:

- **1ターン = 実時間 約2–3秒**
- 1ターン内に被験者は **最大2アクション** を宣言可能（例: 「左手で右袖を狙う」+「腰を左に切る」）
- GMは1ターン内に **1回の防御反応** を宣言（例: 「相手は袖を引いて弾く」）
- ターン終了時に状態ボードを更新

この圧縮により「リアルタイム性」は失われるが、**判断の順序と条件成立の論理**は保存される。M0で検証したいのは後者であるため許容する。

## 2. 物理マテリアル

### 2.1 状態ボード（A3用紙、1枚）

被験者から見た景色を上から見下ろした模式図:

```
┌─────────────────────────────────────┐
│  [相手の姿勢崩し値: 0 ─ 1 ─ 2 ─ 3 ─ 4]   │  ← トークンを置く
│                                       │
│   ┌──────────────┐                    │
│   │ 相手の上半身   │   グリップゾーン:  │
│   │              │   襟L  襟R          │
│   │  [姿勢方向↕] │   袖L  袖R          │
│   │              │   手首L 手首R       │
│   └──────────────┘   帯               │
│         ▲                              │
│   ┌──────────────┐                    │
│   │ 自分の腰     │  腰角度: -2 -1 0 +1 +2 │
│   │              │  押し引き: -2 -1 0 +1 +2│
│   └──────────────┘                    │
│                                       │
│  [左フック: LOCKED] [右フック: LOCKED] │  ← フリップカード
└─────────────────────────────────────┘
```

**離散化のスケール**:
- 姿勢崩し値: 0(完全に立ってる) 〜 4(ほぼ崩れている)。入力系v1.0の `0.0–1.0` を5段階に量子化
- 腰角度: -2(右に大きく切る) 〜 +2(左に大きく切る)。0が正対
- 押し引き: -2(強く引き寄せ) 〜 +2(強く押し離し)
- グリップ強度: 各グリップごとに 0(なし) / 1(軽く) / 2(強く) の3段階

### 2.2 トークン

| トークン | 個数 | 用途 |
|---|---|---|
| 赤丸（被験者の手） | 2 | 左手・右手の現在位置 |
| 青丸（相手の手） | 2 | GM管理 |
| 数値マーカー | 各種 | 姿勢崩し値、腰角度、押し引き、グリップ強度 |
| フックカード | 4 | LOCKED/UNLOCKED 表裏 × 2足 |

### 2.3 技カード（6枚、被験者には伏せない）

入力系v1.0 §D.1.1 の6技を1枚ずつカード化。各カードに発動条件を平易な日本語で記載:

```
┌─────────────────────┐
│  ハサミスイープ        │
│                     │
│  条件:                │
│  ・両足フック LOCKED   │
│  ・左右どちらかの袖を   │
│    強く握る (強度2)    │
│  ・相手の姿勢崩し値≥2  │
│                     │
│  決め手:               │
│  ・腰を真横に切る +    │
│   足フックを片方解放    │
└─────────────────────┘
```

**設計意図**: 条件をカード化して見せるのは、被験者が「条件を意識した上で操作する」体験をさせるため。M0は判断空間の妥当性検証であり、暗記課題ではない。

### 2.4 GM裁定表（A4両面）

§4 のステートマシン遷移表と§5の判断窓判定フローを印刷したもの。GMが手元で参照する。

## 3. セッション進行（15分／1人）

| 時刻 | 内容 | 担当 |
|---|---|---|
| 0:00–1:00 | 趣旨説明、同意取得（録音可否含む） | GM |
| 1:00–3:00 | ボード・トークン・技カードの説明 | GM |
| 3:00–4:00 | ウォームアップ: 「左手で右袖を狙ってください」など単発操作の練習 | 被験者 |
| 4:00–11:00 | **本セッション**: 初期状態から開始し、判断窓が3回発火するか相手がパスするまで | 両者 |
| 11:00–14:30 | 半構造化インタビュー（§7） | GM |
| 14:30–15:00 | 謝辞、次回案内 | GM |

**初期状態**（被験者は攻め側、クローズドガード）:
- 両足フック `LOCKED`
- 全グリップ なし
- 姿勢崩し値 1（軽く前傾）
- 腰角度 0、押し引き 0

## 4. ステートマシン遷移表（GM用）

### 4.1 グリップ試行の裁定

被験者が「左手で◯◯を狙う、強度N」と宣言した時:

| 条件 | GMの宣言 | ボード操作 |
|---|---|---|
| ターゲットゾーンが空き、相手の同所防御なし | 「掴めた」 | 赤丸を当該ゾーンに置く、強度マーカーをN |
| ターゲットゾーンを相手が防御中 | 「弾かれた、手が戻る」 | 赤丸はニュートラル、§4.2 短期記憶を起動 |
| 直近1ターン以内に同手・同ゾーンで弾かれた | 「もう一度弾かれる」 | 同上 |

### 4.2 短期記憶

弾かれたグリップ試行は **次の1ターンの間** 同手・同ゾーンの再試行が無効化される。GMはサイコロまたはチェックマークで記録。

### 4.3 相手AIの行動方針（GM用）

GMは以下のいずれかを毎ターン1つ宣言する。優先順位は上から:

1. **被験者の判断窓条件を1つ崩す**: 例: 被験者が両袖を握っている → 1本を切りに行く
2. **自分の崩れを戻す**: 姿勢崩し値が3以上 → -1し、被験者の腰角度を0に押し戻す
3. **パスを試みる**: 5ターン経過 or 姿勢崩し値0が3ターン継続したら宣言。M0ではパス成立=セッション終了

### 4.4 状態の自然変化

ターン終了時にGMが以下を更新:

- 被験者が腰を切り続けている（同方向 ±1以上を3ターン）→ 姿勢崩し値 +1
- 被験者がグリップなし状態が2ターン継続 → 姿勢崩し値 -1
- 被験者が `BTN_BREATH` 相当（口頭宣言「呼吸を整える」）→ 次ターンの行動権 -1、ただしスタミナ系は今回未実装のため演出のみ

## 5. 判断窓判定フロー（GM用）

### 5.1 発火チェック

各ターン終了時、GMは6技カードを見て、条件がすべて成立しているものをチェック:

```
for each 技カード:
    if すべての条件成立:
        判断窓発火を宣言
        該当技カードを被験者の前に並べる
        「判断窓が開きました。技を選びますか? 見送りますか?」
```

**複数技が同時成立した場合**: すべて並べて被験者に選ばせる。これ自体が判断空間の質を測る重要な観察ポイント。

### 5.2 被験者の応答

| 応答 | GMの処理 |
|---|---|
| 技を選ぶ | 「技が極まりました」→ セッション一旦中断、§7.1の即時インタビューへ |
| 見送る | 判断窓を閉じる。次ターンへ。クールダウン（次ターンは判断窓発火しない） |
| 迷って無回答 | 5秒待って「見送り」扱い |

### 5.3 連発抑制

判断窓発火 → 見送り の後は **次ターン1回スキップ**。入力系v1.0 §D.3 の400msクールダウンに対応。

## 6. 計測項目

### 6.1 セッション中ログ（GM記録）

被験者ごとに以下を記録:

| 項目 | 記録形式 |
|---|---|
| 各ターンの被験者宣言 | 文字列ログ |
| 各ターンのGM裁定 | 同上 |
| 判断窓発火回数 | 数値 |
| 発火時に同時成立した技数 | 数値（複数候補だった割合を後で集計） |
| 技選択 / 見送り の比 | 数値 |
| セッション終了理由 | enum: 技極まる / パスされる / 時間切れ |

### 6.2 M0ゲート判定基準

インタビュー（§7）で以下の3問すべてに「はい/概ねはい」で答えた被験者を **「判断空間を本物と認めた」** とカウントする:

- **Q-A**: 6技の発動条件は、あなたが実戦で技を狙う時の前提条件と一致していましたか?
- **Q-B**: 「条件を作って待つ」という操作感は、あなたが実戦で考えていることと一致していましたか?
- **Q-C**: 判断窓が開いた瞬間、なぜ開いたか自分で説明できましたか?

3問すべてYes = 1票。1問でもNo = 0票。

**ゲート**: §0 のとおり 青帯3/3 かつ 白帯2/3。

---

## 7. インタビュー設計（半構造化、3.5分）

### 7.1 即時質問（技が極まった直後 or セッション中断時、30秒）

- 「今、なぜその技を選びましたか?」
- 「条件が揃った瞬間、自分でわかりましたか?」

### 7.2 セッション後質問（3分）

§6.2 の Q-A / Q-B / Q-C を順に問う。各質問に対し:

1. はい / 概ねはい / いいえ で回答
2. **「いいえ」または「概ねはい」の場合、具体的にどこが違ったか自由回答**

最後に自由質問:
- 「もし1つだけ技の条件を変えるとしたら、どれをどう変えますか?」
- 「『条件を作って待つ』以外に、柔術で大事な意思決定はありますか?」（M1 以降の入力モデル拡張のヒント収集）

## 8. 役割分担と運営

### 8.1 GM（1名、固定）

- セッション中の裁定、相手AI役、ボード操作、ログ記録
- **被験者によって裁定にブレが出ないよう、GMは全6セッションを通じて同一人物**とする
- GMは事前に §4・§5 をリハーサル（1人で2セッション分）

### 8.2 観察者（1名、推奨）

- 録音・録画機器の管理
- インタビュー中のメモ取り（GMの裁定負荷を下げる）
- 任意。確保できなければGM兼任

### 8.3 物理セットアップ

- 道場の畳の上、または壁際に小机を置きボードを広げる
- 被験者は道着姿のままで参加可（実戦感を保つため）
- 録音は同意が取れた被験者のみ

## 9. 実施スケジュール

| 日程 | 内容 |
|---|---|
| 実施1週前 | GMリハーサル、マテリアル印刷、被験者リクルート最終確認 |
| 実施日 | 6セッション（1人15分 + 設営/休憩 = 計約3時間） |
| 実施翌日 | ログ整理、ゲート判定 |
| 実施1週後 | 本文書 v1.1 への改訂（被験者フィードバック反映） |

**合格時**: M1（IKグリップファイト実装）に進む。
**不合格時**: どの帯のどの質問が落ちたかを分析し、入力系v1.0 を改訂してM0再実施。
